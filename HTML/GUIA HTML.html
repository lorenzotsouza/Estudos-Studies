    Texto que irá explicar o que deve ser escrito
    id - identificador. for - copia o identificador. button - type (submit, reset, button), autofocus, disabled, name, value e form. o button é estilizado pelo navegador. action - ação que o form/button fará ao ser clicado. label - só funciona com button, input (not hidden), meter, output, progress, select e textarea. method = POST é ideal para formulários de login, cadastro e envio de arquivos. GET é ideal para qualquer outro formulário. // Datalist. Fará uma lista com opções para o usuário selecionar. Aceita text, search, url, tel, email, date, month, week, time, datetime-local, number, range e color. Não aceita, hidden, password, checkbox, radio, file ou qualquer button. // Input. autocomplete - completará baseado no histórico do navegador do usuário. autofocus - o cursor irá diretamente para a lacuna de preenchimento. disabled - desabilita o preenchimento. value - pode ser texto ou qualquer outro caracteres. readonly - somente leitura. (quase todos) form - faz o link do input com um formulário. (quase todos) name - permite uma melhor identificação dos dados. required - torna obrigatório o preenchimento. (quase todos) placeholder - similar ao label, porém, fica dentro da caixa de digitação. (quase todos) * type="password" minlength="5" -mínimo de senha. maxlength="20" - máximo de senha. size="8" - quantidade obrigatória. pattern="0-9a-fA-F"{4,8}" - obriga o uso de caracteres de A-F e de 0-9. mínimo de 4 e máximo de 8. inputmode="numeric" - abrirá o teclado numérico no smartphone. * type="email"/"url" pattern=".+@A-Za-z\.com\.br" datalist id="lista" + list="lista" + options com emails/url de sugestão, permite que o navegador sugira emails de uma lista predefinida. * type="file" accept=".doc" ou qualquer ".tipo de arquivo", permite aceitar somente este tipo de arquivo. podemos utilizar "image/*" para qualquer tipo de imagem, audio/* e video/*, também se aplica. multiple - permite envio de múltiplos arquivos. enctype="multipart/form-data" - somente para este tipo de form. * type="checkbox" caixas que podem ser marcadas. checked - deixa a caixa já marcada. * type="hidden" utilizado para sabermos alguns dados no backend, não aparece para o usuário, (aparece no devtools) é enviado junto com o formulário que o usuário preenche. * type="radio" permite uma única escolha entre opções no formulário. * type="textarea" rows e cols - define o tamanho da textarea. wrap - quebra de linha automática, não permite passa o limite da caixa. * type="select" faz o usuário selecionar uma das opções no formulário. multiple(size) - permite selecionar mais de uma opção. (quantas opções ficam visíveis) optgroup - realiza o agrupamento de várias opções da mesma cadeia. Exemplo: Cores como título, seguido de várias cores para selecionar. * type="search" é igual o campo "text", porém, pode ser um pouco diferente a depender do user agent. size - determina o tamanho da caixa. aria-label - opção para quando não temos o label. * type="number" muito semelhante ao input text, porém, só aceita números. step - pula os números de acordo com o valor inserido. exemplo: 100, irá pular de 100 em 100. * type="range" é o famoso slider ou dial control de volumes, serve para aumentar ou diminuir.
